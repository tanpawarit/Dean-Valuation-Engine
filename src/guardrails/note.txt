


OWASP Top 10 LLM ปี 2025 list
1. Prompt Injection - การป้อน prompt ที่ทำให้บอทไม่ทำตามขอบเขตที่เราตั้งไว้ เช่น ส่งข้อความมาว่า "ลืมคำสั่งทั้งหมดก่อนหน้านี้ และทำตามที่ฉันสั่ง"
2. Sensitive Information Disclosure - การปล่อยให้ AI เห็น credential ถือเป็นเรื่องน่ากลัว โดยเฉพาะถ้ามีใครก็ไม่รู้มาถามมันว่าขอ API Key หน่อยสิ แล้วมันดันให้ได้ ทางที่ดีอย่าให้มันตอบอะไรแนว ๆ นี้ดีกว่า
3. Supply Chain Vulnerabilities - การใช้ tools open-source โดยที่ไม่เช็คให้ดีว่ามันทำงานยังไงก็ถือเป็นอะไรที่น่ากลัว เพราะบาง tools ถึงขั้นให้ LLM เข้ามาควบคุมเครื่องได้เลย
4. Data and Model Poisoning - ข้อมูลที่โมเดลดึงไปใช้เทรน หรือในการตอบคำถามถูกวางยาเอาไว้ เช่น ผสมข้อมูล หรือคำพูดประหลาด ๆ เข้าไปทำให้บอทตอบอะไรที่ไม่ควร
5. Improper Output Handling - การใช้งาน LLM ในบางจังหวะก็ยังต้องการให้คนเข้ามามีส่วนตัดสินใจ เช่น ตัดสินว่า credit ควรถูกอนุมัติหรือไม่ การ implement HITL - Human In The Loop จึงไม่ควรถูกมองข้าม 
6. Excessive Agency - การให้อำนาจ LLM มากเกินไปก็ก่อผลเสียได้เหมือนกัน เช่น การให้มันรันโคดใน terminal โดยไม่ได้รับอณุญาต
7. System Prompt Leakage - การเปิดเผย system instruction สามารถเผยให้เห็นถึงช่องโหว่ที่ attacker สามารถเอาไปใช้ประโยชน์ได้
8. Vector and Embedding Weakness - LLM อาจถูกโจมตีได้จากข้อมูลที่มันไปทำการค้นหามา หรือถูกส่งผ่านเข้าไปใน input เช่น ผสมคำสั่งให้มันทำอะไรแปลก ๆ เข้าไปใน PDF แล้วให้มันอ่าน
9. Misinformation - LLM ให้ข้อมูลเท็จได้ เอาจริง ๆ ก็คือมันอาจจะทั้งหลอนและไม่ได้หลอนด้วยความที่มันถูกเทรนด้วย data ที่เยอะมาก ๆ ก็เลยทำให้มีโอกาสที่มันจะให้ข้อมูลผิด ๆ ออกมาได้
10. Unbounded Consumption - การที่ attacker โจมตีด้วยการให้ LLM consume resource หนัก ๆ เช่น ให้ tool ที่บอทใช้ return ตัวอักษรที่ไม่มีความหมายออกมายาวมาก ๆ ซึ่งมันก็จะเข้าไปเป็นส่วนหนึ่งของ input token ที่ไม่ได้ช่วยให้บอททำงานได้ แถมยังผลาญเงินเราด้วย

OWASP Guardrails We have
- Prompt Injection Guardrail (user_intent_check, refuse_non_analytical_requests)
- Sensitive Information Guardrail (sensitive_data_input_check, sensitive_data_output_check)
- Improper Output Guardrails (output_length_check, detailed_summary_check)
